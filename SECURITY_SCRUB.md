# Очистка истории от секретов (маскировка) — локально и на GitHub

В истории git были захардкожены секреты: зашифрованные объекты `gAAAA…` (Fernet) и
плэйнтекст master key — в `front.py` и `mcp_server.py`, практически в каждом коммите.
Удаление из рабочего дерева НЕ убирает их из истории: любой `git log`/клон их покажет.

Сценарий, под который написана инструкция: **один локальный клон, работа велась только в нём,
нужно вычистить его и перезаписать GitHub.** Повторно клонировать из GitHub не требуется.

> ⚠️ Порядок важен. Шаг 0 (ротация) — обязателен и не заменяется маскировкой.

---

## 0. Сначала — РОТАЦИЯ (секреты считать скомпрометированными)

Эти значения уже были на GitHub: их могли склонировать, форкнуть, закэшировать, проиндексировать.
Маскировка истории НЕ отменяет утечку. Поэтому до/параллельно со scrub:

- сгенерировать новый **master key** и новый **Fernet-ключ**, перешифровать `db_conf` и storage key;
- сменить **пароль/креды БД**, **keycloak client secret**, любые токены/ключи источников, что шифровались;
- инвалидировать выпущенные API-ключи приложения при сомнениях.

Без ротации scrub лишь «приберёт» репозиторий, но утёкшие секреты останутся валидными.

---

## 1. Убедиться, что рабочее дерево чистое и очистка закоммичена

Хардкод уже убран (`front.py` — `pwinput`/аргументы; `mcp_server.py` — окружение/аргументы) и
закоммичен. Проверка — обе команды должны вернуть пусто:

```
cd /Users/***REMOVED***/Documents/dev/UH2S
git status --short          # нет незакоммиченных изменений
git grep -I -c "gAAAA" -- . # секретов в рабочем дереве нет
```

Пушить сейчас НЕ нужно: scrub перепишет всю историю, включая последний коммит, а затем мы
перезапишем GitHub целиком одним force-push.

---

## 2. Сделать резервную копию локального репозитория (обязательно)

Операция необратима. `--mirror` копирует репозиторий целиком (все ветки и теги):

```
git clone --mirror . ../UH2S-backup.git
```

Если что-то пойдёт не так — восстановите из `../UH2S-backup.git` (см. раздел «Откат»).

---

## 3. Установить git filter-repo

```
pip install git-filter-repo        # либо: brew install git-filter-repo
git filter-repo --version
```

---

## 4. Подготовить правила замены

```
cp secrets-replacements.txt.example secrets-replacements.txt
```

Откройте `secrets-replacements.txt` и при необходимости дополните:
- строка `regex:gAAAA…` уже ловит **все** Fernet-токены во всей истории автоматически;
- добавьте `literal:`-строки с **реальными** значениями плэйнтекст-секретов (старый master key и т.п.),
  каждое в своей строке: `literal:СТАРЫЙ_MASTER_KEY==>***MASTER_KEY_REMOVED***`.

Файл `secrets-replacements.txt` уже в `.gitignore` — не коммитьте его.

---

## 5. Переписать историю ПРЯМО в локальном репозитории

Так как чистим не «свежий клон», а рабочий репозиторий, добавляем `--force` (бэкап из шага 2 страхует):

```
cd /Users/***REMOVED***/Documents/dev/UH2S
git filter-repo --replace-text secrets-replacements.txt --force
```

filter-repo пройдёт по содержимому всех коммитов/веток/тегов и заменит секреты на заглушки.
**SHA коммитов изменятся** — это нормально (вы единственный, кому нужен этот репозиторий).

Проверка, что в истории больше нет токенов (должно быть пусто):

```
git log --all -p -S "gAAAA" | head
```

---

## 6. Вернуть origin и перезаписать GitHub

filter-repo намеренно удаляет remote `origin` (чтобы вы не запушили смесь старого и нового
по ошибке). Добавляем заново и пушим с перезаписью:

```
git remote add origin https://github.com/orbitalrailgun/UH2S.git
git push --force --all
git push --force --tags
```

После этого ваш локальный репозиторий и GitHub синхронны и без секретов. Переклонировать
ничего не нужно — это и есть ваш рабочий клон.

---

## 7. Очистка на стороне GitHub

Force-push НЕ удаляет старые коммиты мгновенно — они остаются доступны по прямому SHA из кэша,
пока не пройдёт сборка мусора, и могут жить в форках/PR. Поэтому:

- удалить/уведомить про форки; закрыть PR, ссылающиеся на старые коммиты;
- открыть тикет в **GitHub Support** с просьбой очистить кэш недоступных (stale) коммитов;
- убедиться, что секреты из шага 0 **ротированы** — это единственная настоящая защита.

---

## Откат (если маскировка прошла неверно)

Восстановить локальный репозиторий из бэкапа шага 2:

```
cd ..
rm -rf UH2S
git clone UH2S-backup.git UH2S
cd UH2S
git remote set-url origin https://github.com/orbitalrailgun/UH2S.git
```

(GitHub откатывать не придётся, если вы ещё не делали force-push.)

---

## Альтернатива без filter-repo — BFG

```
bfg --replace-text secrets-replacements.txt .git
git reflog expire --expire=now --all && git gc --prune=now --aggressive
```

---

## Памятка

- Делайте бэкап (шаг 2) до запуска filter-repo — откатить иначе нечем.
- Теги v0.5.0…v0.8.x будут перезаписаны — это нормально.
- `secrets-replacements.txt` и `*-backup.git/` — не коммитить (в `.gitignore`).
- Главное — РОТАЦИЯ секретов (шаг 0); маскировка истории её не заменяет.
